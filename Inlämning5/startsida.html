Skillnaden mellan inline CSS och extern stilmall

Det finns flera olika sätt att styla ett HTML dokument, bland annat inline style och extern stilmall.

Inline style är när man sätter ett style element på varje html tagg man vill styla. Inom de style elementen skriver man den CSS kod men vill styla HTML koden med. Man kan skriva hur många attribut man vill inom style elementet, det finns ingen begränsning.

Fördelen med inline style är att man kan vara väldigt specifik för vilket HTML tagg man vill styla. Den är också bra att använda när man ska stylar med Javasqript som ofta använder inline style. Nackdelen med inline style är att det lätt kan bli rörigt att ha CSS kod och HTML kod tillsammans i samma dokument och det kan bli krångligt att ändra i efterhand. Äldre webbläsare kan till och med tolka CSS koden som html kod och därav visa upp det direkt på webbsidan.

Med external style sheet så har man ett separat CSS dokument där man stylar ẃebbsidan. Det betyder att i alla html dokument har man ett link element i head taggen där man länkar till CSS dokumentet (alla sidor länkar till samma dokument). Det betyder att all styling för alla HTML sidor hamnar i samma dokument.

Fördelen med en extern stilmall är att man samlar all styling på samma ställe vilket gör det enkelt att återanvända CSS klasser till alla html dokument som är kopplade till CSS dokumentet. Det blir också renare och snyggare att separera HTML kod och CSS kod. Nackdelen med en extern stilmall är att om man stylar alla HTML sidor i samma css dokument kan det dokumentet bli väldigt långt och rörigt.
